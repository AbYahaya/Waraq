# WARAQ — CANONICAL USER-STYLE CORPUS v1.0

## Document A — content basis for the feature "Recognize my translation style"

No code. No coding release. No silent re-baselining. This document is not the finished feature specification but the substantive style basis on which the later structured feature specification must build.

## 1. ROLE OF THIS DOCUMENT

This document defines the canonical translation style of the user for Arabic → German in a form that can later:

- enter the structured feature specification "Recognize my translation style",
- be integrated into the further Waraq implementation track,
- not overwrite the already-released general system rules.

This document is a building block within the Waraq implementation track.

## 2. PRECEDENCE LOGIC (CENTRAL)

The individual user style is an additional layer, but not a counter-layer to already-released general rules.

### 2.1 Order of validity

In the case of conflict the following holds:

- already-released general system rules
  - Transliteration
  - Glossary
  - Indexes
  - Terminology
  - Religious formulas
  - Other already-released canonical standards
- canonical user style
  - this document
- individual reference sentences
  - as style evidence
  - not as silent replacement for general system standards

### 2.2 Important consequence

If a reference sentence contains a spelling, transliteration, or terminology that contradicts the already-released system rule, then:

- the system rule retains precedence
- the reference sentence remains stylistically authoritative
- the concrete spelling is productively adapted to the canonical system rule

### 2.3 Consequence for the style-feature application

The style-feature application must not integrate this user style in such a way that already-released:

- transliteration,
- glossary rules,
- index logic,
- terminology standards

are undermined or silently replaced.

## 3. SCOPE OF THE USER STYLE

This style applies in particular to:

- classical Islamic texts
- scholarly / specialist-language texts
- polemical / argumentative texts
- texts close to:
  - Qurʾān
  - Ḥadīṯ
  - Isnād
  - ʿIlal
  - Fiqh
  - Uṣūl
  - Balāgha
  - ʿAqīda

It is:

- account-bound
- user-specific
- not global
- not standard for new accounts
- only activatable after sufficient self-confirmed material

## 4. HARD STYLE INVARIANTS

These rules apply at highest priority.

### 4.1 Absolute word-fidelity

- Every word should be translated where possible.
- No silent omissions.
- No silent simplifications.
- No summaries in place of translation.

### 4.2 Structural proximity to the Arabic

- Sentence movement to be kept as close as possible to the Arabic.
- Order of statements to be preserved as far as possible.
- Arabic chains of connection not to be unnecessarily smoothed.
- Mirroring sentence flow is expressly desired, as long as the German sentence remains carriable.

### 4.3 Repetition stays

- Repetitions may not be removed for readability reasons.
- Doublings, parallelisms, and intensifications remain visible.

### 4.4 No flattening of religious and specialist terms

- Specialist terms are kept precise.
- Religious terms are not softened.
- Polemical / evaluative terms are not modernized or defused.

### 4.5 No silent explanations

- The translation does not explain on the sly.
- If minimal comprehension safeguarding is necessary, this happens openly through parentheses.

### 4.6 No modern or casual German

- no journalistic tone
- no casual style
- no simplified everyday German
- target register: elevated, text-near, classical

## 5. STRONG PREFERENCES (NOT MECHANICAL, BUT CLEAR)

### 5.1 Maximum word-fidelity wins "almost" always

The user style prefers maximum word-fidelity almost always, but not blindly mechanically. Confirmed reference sentences may show that a slightly precision-enhancing solution is better than raw mechanics.

### 5.2 Sentence structure close to the Arabic movement

- Arabic sentence movement should remain visible
- but not so strongly that the German sentence becomes uncarriable

### 5.3 Co-translate pronouns where possible

- Pronouns and references should be preserved
- where needed, brief reference resolution in parentheses

### 5.4 Parentheses are expressly permitted

Parentheses are not only an emergency solution but a deliberate means for:

- term-precisification
- reference resolution
- brief comprehension safeguarding
- work reference
- metaphorical safeguarding

### 5.5 Specialist precision may overtake raw word-literalness

If an expression is clearly specialist, specialist precision may overtake raw dictionary-proximity without losing text-proximity.

## 6. STRUCTURED STYLE FEATURES

### 6.1 Lexical level

- translate every word where possible
- keep terminology consistent
- no random synonym changes for specialist terms

### 6.2 Syntactic level

- closely mirror Arabic structure
- keep connectors with "Und", "so", "dann", "vielmehr", "denn" visible
- do not unnecessarily recompose sentence movement

### 6.3 Rhetorical level

- repetitions stay
- intensity stays
- parallelisms stay
- imagery stays
- exaggeration / pointing-up stays

### 6.4 Specialist-language level

- ḥadīṯ-technical language stays technical
- juridical-contractual language stays juridically visible
- polemical language stays polemically visible
- grammatical-rhetorical terms are handled precisely

### 6.5 Parenthesis policy

Permitted are brief parentheses for:

- term-precisification
- reference resolution
- metaphorical safeguarding
- work reference
- minimal comprehension safeguarding

Not permitted:

- sprawling commentary
- free interpretation in parentheses
- paraphrase instead of translation

## 7. PARTICULARLY RELEVANT PHENOMENON FIELDS FOR THE LATER FEATURE

These fields must be modeled in structured form in the later style-recognition feature:

- particle handling
- sentence connection / repetition
- Idāfa handling
- Masdar / verb relation
- specialist equivalences
- handling of Qurʾān and Ḥadīṯ citations
- handling of Isnād / ḥadīṯ-critical specialist language
- parenthesis use
- religious-polemical terms
- juridical-contractual metaphor
- register height
- errors that must not happen again

### Reference sentence 5

**Arabic original**

**Kanonische deutsche Endfassung**

Und derjenige, den die Nachricht von der Mühsal der Muḥaddiṯūn hinsichtlich der verborgenen Mängel (ʿilal) des Ḥadīṯes nicht erreicht hat, fragt sich und sagt: Warum hat er das, womit Muslim allein blieb (was Muslim allein überlieferte), dem vorgezogen, worauf die beiden sich einigten, und sich nur mit einem Hinweis darauf begnügt, obwohl der Wortlaut des Ḥadīṯes einer ist (einer und derselbe), und die Überlieferungskette in beiden zusammenhängend (verbunden bzw. lückenlos) ist? Und die Antwort ist: Im Ḥadīṯ von Ibn ʿAbbās, worüber Einigkeit (zwischen al-Buḫārī und Muslim) besteht, ist ein verborgener Mangel, der nicht sichtbar ist ausser für denjenigen, der Erfahrung in der Untersuchung der Überlieferungskette hat, und dieser ist: dass der Ḥadīṯ von Ibn ʿAbbās – Allahs Wohlgefallen auf ihm – von Manṣūr, über Muǧāhid, über Ṭāwūs, über Ibn ʿAbbās, über den Propheten ﷺ, zusammenhängend (mit verbundener bzw. lückenloser Überlieferungskette) überliefert wurde. Dies ist das, was in den beiden Ṣaḥīḥ-Werken vorhanden ist. Und al-Aʿmaš widersprach ihm (Manṣūr), und so überlieferte er ihn über Muǧāhid, über den Propheten ﷺ als Mursal(-Ḥadīṯ). Saʿīd ibn Manṣūr überlieferte ihn über Abū Muʿāwiya, über ihn (al-Aʿmaš). Und ebenso überlieferte er ihn über Sufyān, über Dāwūd ibn Šābūr, über Muǧāhid als Mursal(-Ḥadīṯ), und Manṣūr ist eine vertrauenswürdige Person, ein Ḥāfiẓ, so ist das Urteil für seine Verbindung. Und deshalb bevorzugten die beiden Verfasser des Ṣaḥīḥ seine Verbindung, und überlieferten ihn, aber wegen des Vorhandenseins dieses Unterschiedes überlieferte er ihn nicht und begnügte sich mit einem Hinweis darauf. Und dies, wie du siehst, wird nicht erfasst ausser durch das Nachverfolgen der Überlieferungswege des Ḥadīṯes, insbesondere da sein Vorhandensein in den beiden Ṣaḥīḥ-Werken ein Anlass dafür ist, dass ihm keine Aufmerksamkeit gegeben wird.

**Extracted style signals**

- scholarly specialist language stays technical
- parentheses are strongly permitted in ḥadīṯ-critical texts
- isnād language is not popularized
- terminological consistency on specialist verbs is important
- "ثقة" rendered as a person-noun rather than merely adjectivally
